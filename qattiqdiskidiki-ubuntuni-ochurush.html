نىمۇ ئۇبۇنتۇنى تۈزۈك تەشۋىق قىلە قىلماي ئۆچۈرۈش ئۇسۇلىنى سۆزلەپ يۈرگەندۇ دەپ قالماڭلار. ئۇبۇنتۇ بىلەن Windows نى قوش سىستېما قىلىپ يېڭى ئىشلەتكۈچىلەر، تۆۋەندىكىدەك ئىككى مەسىلىگە يولۇقۇشى مۇمكىن.

  • ئۇبۇنتۇغا كۆنەلمەي ئۇبۇنتۇ قاچىلىغان دىسكا رايونىنى فورماتلىۋېتىپ Windows قىمۇ كىرەلمەسلىك.
  • ئۇبۇنتۇنى قايتا قاچىلاپ، سىستېما قوزغالغاندا Windows نى تاللاش تۈرى كۆرۈنمەسلىك (ئەلۋەتتە بۇنى ئۇبۇنتۇغا كىرىپ تۇرۇپ ھەل قىلىش ئانچە تەس ئەمەس).

ئەمەلىيەتتە ئۇبۇنتۇ بىلەن Windows نى قوش سىستېما قىلغاندا، Windows نى ئالدىدا ئۇبۇنتۇنى كەينىدە قاچىلىساق ئەڭ ياخشى. كەينىدە قاچىلىغان ئۇبۇنتۇ grub بولسا Windows نىڭ MBR سىنىڭ ئۈستىگە قاچىلىنىدۇ (يەنى MBR نى بۇزۇۋېتىدۇ)، شۇڭا قوش سىستېما قىلغاندا بىراقلا ئۇبۇنتۇ دىسكا رايونىنى فورماتلىساق grub مۇ بۇزۇلۇپ كەتكەچكە بۇرۇن بۇزۇلۇپ بولغان MBR بولسا Windows نى ئەسلا قوزغىتالمايدۇ.
شۇڭا ئەڭ ياخشىسى ئۇبۇنتۇنى ئۆچۈرۈشتىن ۋە ئۇبۇنتۇ قايتا قاچىلاشتىن بۇرۇن mbrfix دېگەن يۇمشاق دېتال بىلەن Windows نىڭ بۇزۇلغان MBR نى ئوڭشاش كېرەك.
‏mbrfix نى‏ چۈشۈرۈۋېلىڭ:

‏

mbrfix نى ئىشلىتىش ئۇسۇلى:
‏ئالدى بىلەن چۈشۈرگەن mbrfix.exe نى C دىسكىنىڭ غول مۇندەرىجىسىگە قويىمىز. ئاندىن ‏运行‏ گە كىرىپ cmd بۇيرۇقىنى كىرگۈزىمىز. cd بۇيرۇقىنى يۈرگۈزۈش ئارقىلىق C دىسكىنىڭ غول مۇندەرىجىسىگە كىرىمىز. mbrfix /drive 0 fixmbr /yes كىرگۈزۈپ ئېنتىر ئۇرساقلا Windows نىڭ MBR ئەسلىگە كېلىدۇ (مەن Windows7 دىلا سىناپ باقتىم).
بۇ ئۇسۇل ھەقىقەتەن ئاددىي. كۆپچىلىكنىڭ قوش سىستېما شارائىتىدا ئۇبۇنتۇ ئۆچۈرۈش ۋە قايتا قاچىلاشتا بۇ ئۇسۇلنى قوللىنىشى ئۇمىد قىلىمەن.
