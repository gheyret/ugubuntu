قوش سىستېما شارائىتىدا Win7 نى قايتا قاچىلىغاندىن كىيىن ئۇبۇنتۇ 10.10 غا كېرەلمەسلىك مەسىلىسىنى ھەل قىلىش
ئېلان قىلىنغىلى420 كۈن بولدى / ئاساسى ساۋات / 10 پارچە ئىنكاس چۈشتى / كۆرۈلگىنى 908+

ئۇبۇنتۇ بىلەن Win7 بىرگە ئىشلىتىۋاتقانلار بىلسە كېرەك. Win7 قايتا قاچىلىغاندىن كىيىن ئۇبۇنتۇغا كىرگىلى بولمايدۇ. مەن تۆۋەندە ئۇبۇنتۇ 10.10 دا بۇ مەسىلىنى ھەل قىلىش ئۇسۇلىنى تونۇشتۇرۇپ ئۆتەي. ئۇبۇنتۇنىڭ باشقا نۇسخىلىرى ئوتتۇرىسىدا ئازراق پەرق بولۇشى مۇمكىن بىراق ئاساسەن ئوخشاش.
بۇنى ھەل قىلىشنىڭ مەن بىلىدىغان ئىككى خىل ئۇسۇلى بار.

  • ئۇبۇنتۇنى قوزغىتىش تىزىملىكىنى Windows دىكى boot قا قوشۇش (WinXP دە ئاسانراق، بىراق Win7 دە مەن قىلالمىدىم شۇڭا بۇ يەردە تونۇشتۇرمايمەن).
  • ئۇبۇنتۇنىڭ LiveCD سى بىلەن ئوڭشاش

ئالدى بىلەن كومپيۇتېرنى ئۇبۇنتۇ 10.10 نىڭ CD دىسكىسىدىن قوزغىتىمىز. ھەم تېرمىنالنى ئېچىپ تۆۋەندىكى ئىككى بۇيرۇقنى تەرتىپ بويىچە كىرگۈزىمىز.

ubuntu@ubuntu:~$sudo　-i
root@ubuntu:~#fdisk -l

تېرمىنالدا كۆرۈنگەن ئۇچۇرغا قاراپ ئۇبۇنتۇرۇمىزنىڭ قايسى رايونغا قاچىلانغىنى كۆرۈۋالىمىز. مەسىلەن / رايون sda6، ئەگەر boot/ مۇ ئايرىم ئايرىلغان بولسا sda7 (ئايرىم ئايرىمىغان بولساق ئىشىمىزنى قىلىمىز، ھى…) تېرمىنالغا تۆۋەندىكى بۇيرۇقنى كىرگۈزىمىز.

root@ubuntu:~#mount /dev/sda6 /mnt
root@ubuntu:~#mount /dev/sda7 /mnt/boot
ubuntu@ubuntu:~#grub-install –root-directory=/mnt/ /dev/sda
ubuntu@ubuntu:~#exit

سىستېمىنى قايتا قوزغىتىمىز. قايتا قوزغاتقاندا grub> چىقىشى مۇمكىن، تۆۋەندىكى بۇيرۇقنى كىگۈزمىز.

ubuntu@ubuntu:~#grub>kernel /boot/grub/core.img
ubuntu@ubuntu:~#grub>boot

ئۇبۇنتۇغا كىرگەندىن كىيىن، 系统System－管理Administration－Synaptic Package Manager新立德软件包 نى ئېچىپ grub-pc نى ئىزدەپ قاچىلايمىز. ئەگەر قاچىلاقلىق بولسا ئۆچۈرۈپ قايتا قاچىلايمىز.
ئەڭ ئاخىرىدا تېرمىنالنى ئېچىپ تۆۋەندىكى ئىككى بۇيرۇقنى يۈرگۈزسەكلا ئىش تامام.

uyghur-archive.com/ugubuntuputer:~$sudo　-i
uyghur-archive.com/ugubuntuputer:~#update-grub2

ئۇبۇنتۇ بىلەن Windows نى بىرگە ئىشلىتىۋاتقانلار، قاتتىق دىسكىغا قاچىلانغان ئۇبۇنتۇنى توغرا ئۆچۈرۈش دېگەن يازمام بىلەن بۇ يازمىنى تەپسىلىي كۆرۈپ چىقسا بىر سىستېما بۇزۇلسا ئىككى سىستېمىنى تەڭ قاچىلاش ئاۋارچىلىقىدىن قۇتۇلىدۇ.

  2. بىغەم : 2011-يىل 5-ئاۋغۇست15:53 جاۋاب

    بۇ ئۇسۇل 11.4كە ماس كەلمەمدۇ قانداق؟

    ubuntu@ubuntu:~$sudo　-i
    root@ubuntu:~#fdisk -l

    بۇ ئىككى بۇيرۇقنى كىرگۈزسەم command not found دەپ ئۇچۇر چىقىۋالدى.

    • Bakhtiyar : 2011-يىل 5-ئاۋغۇست21:58 جاۋاب

      sudo -i بۇيرۇقىنى كىرگۈزۈپ، دەرىجىدىن تاشقىرى ئىشلەتكۈچىگە ئايلىنىپ، ئاندىن fdisk -l بۇيرۇقى بىلەن، كومپيۇتېردىكى ئايرىلغان رايۇننى تاپىسىز، ياكى بىۋاستە sudo fdisk -l دەپ كىرگۈزسىڭىزمۇ بولىدۇ.
      ئاندىن كۆرۈنگەن دىسكا ئۇچۇرىدىن چوڭ كىچىكلىكىگە قاراپ تۇرۇپ، boot نى ئايرىم ئارىغان بولسىڭىز شۇنى، ياكى بىۋاستە غول مۇندەرىجى يەنى / مۇندەرىجىسىنى تېپىپ مەشغۇلات قىلسىڭىز بولىدۇ.

      fdisk -l بۇيرۇقى ئۇبۇنتۇ 10.10 دا ئوپتىك دىسكىدىن قوزغاتقاندا نورمال ئىشلەيتى بولمىسا، شۇنداقلا بۇ بارلىق لىنۇكسدىكى دىسكا ئايرىشتا ئىشلىىلدىغان بۇيرۇق. ئەگەر boot نى ئايرىم ئارىغان بولسىڭىز شۇنى، ياكى بىۋاستە غول مۇندەرىجى يەنى / نىڭ سانى بىلسىڭىز بۇ باسقۇچنى قىلمىسڭىزمۇ بولىدۇ.
