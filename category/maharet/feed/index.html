تەييارلىق ۋە دىققەت قىلىشقا تېگىشلىك ئىشلار

Windows7 دە ئۇبۇنتۇغا يىراقتىن كىرىش ئۈچۈن ئەڭ مۇھىم ئىككى شارائىت ھازىرلىشىمىز كېرەك. بىرى تور مۇھىتى، يەنى ئىككى كومپيۇتېر ئىنتېرنېتقا ئۇلانغان بۆلىشى،  دائىرىلىك توردا (局域网) بولسا تېخى ياخشى، تور سۈرىتى بەك ئاستا بولماسلىقى كېرەك. (ئىككى كومپيۇتېردا ئايرىم ئايرىم IP نى ئۆزئارا ping قىلىپ تەكشۈرۈپ باقساڭلار بولىدۇ). لىنۇكسدا IP ئادرېسىنى كۆرۈش ئۈچۈن ifconfig بۇيرۇقىنى ئىشلىتىمىز.

يەنە بىرى، كىرمەكچى بولغان كومپيۇتېرنىڭ سەپلىمىسى بەك تۆۋەن بولماسلىقى كېرەك. چۈنكى تونۇشتۇرماقچى بولغىنىم، Windows7 دە ئۇبۇنتۇنىڭ گرافىك يۈزىگە يىراقتىن كىرىش ئۇسۇلى، بەزى كونا كومپيۇتېرلاردا بىر ئادەم ئاران مەشغۇلات قىلالايدۇ (سۈرەت ئاستا، قاتىدۇ دېمەكچى)، ئەگەر بۇنداق كومپيۇتېرغا قاچىلانغان ئۇبۇنتۇغا بىرەرى مەن تونۇشتۇرغان ئۇسۇل بىلەن يىراقتىن كىرسە، كومپيۇتېر پارتلاپ كەتمىسۇن  بۇ ئۇسۇل ئالىي مەكتەپ ئوقۇغۇچىلىرىغا تولىمۇ ماس كېلىدۇ. ئادەتتە كارخانا، ئورۇنلاردا  لىنۇكسنىڭ گرافىك يۈزىگە يىراقتىن كىرىش ئۇسۇلى ئاساسەن قوللىنىلمايدۇ، ssh ئارقىلىق خەتلىك تېرمىنالغا كىرىدۇ. بۇنى كېيىنچە ۋاقتىم چىقسا يازارمەن.

تەپسىلىي مەشغۇلات جەريانى

ئالدىن بىلەن ئۇبۇنتۇدا Remote Desktop قورالىنى قوزغىتىمىز، بۇ سىستېما بىلەن بىرگە قاچىلاغلىق، ئۆچۈرۈۋەتكەنلەر بولسا يۇمشاق دېتال مەركىزىدىن ئىزدەپ قاچىلىۋالساڭلار بولىدۇ. بۇ قورال ئارقىلىق باشقىلارنى كومپيۇتېرىمىزغا يىراقتىن كىرەلەيدىغان قىلىپ بەلگىلەيمىز. بۇ يەردىكى مۇھىمى ئەڭ ئۈستىدىكى ئىككى تاللاش. قالغانلىرىنىڭ نېمە ئىكەنلىكگە قىزىقساڭلار سىناپ بېقىڭلار. مەن بەلگە قىلىپ ئۆچۈرۈۋەتكەن يەردە IP رىمىز كۆرۈنىدۇ.

يۇقىرىدىكى مەشغۇلات تاماملانغاندىن كىيىن Windows7 نىڭ بۇيرۇق يۈرگۈزىدىغان يېرىنى (运行) ئاچىمىز، ھەم mstsc.exe نى كىرگۈزۈپ يۈرگۈزىمىز.

ئېچىلغان يېڭى كۆزنەككە كىرمەكچى بولغان ئۇبۇنتۇنىڭ IP ئادرېسىنى كىرگۈزىمىز.

ئاستىدىكى ئۇلاشنى باسساق مۇۋەپپەقىيەتلىك بولسا تۆۋەندىكىدەك كۆزنەك كۆرۈنىدۇ، ئۇبۇنتۇدا بار بولغان قوللانچى نامى، شىفىرىنى كىرگۈزۈپ كىرسەكلا بولىدۇ.

گەپنى ئۇزارتىپ مۇشۇ يەرگە ئەكەپتىمەن. ئەمدىن Windows7 دە باشقا Windows نىڭ  گرافىك يۈزىگە يىراقتىن كىرىشمۇ يۇقىرىدا تونۇشتۇرغىنىم بىلەن كۆپ پەرقلەنمەيدۇ. كىرمەكچى بولغان  Windows نى باشقىلار يىراقتىن كىرەلەيدىغان قىلىپ بەلگىلەيمىز، ئاندىن قالغان مەشغۇلات ئوخشاش. يەنى كومپيۇتېرىم –› خاسلىقىغا (我的电脑，属性) كىرىپ رەسىمدىكىدەك قىلىمىز.

32 بىتلىق مەشغۇلات سىستېمىسى، 64 بىتلىق مەشغۇلات سىستېمىسى ھەققىدە سۆزلەشتىن بۇرۇن، بۇلارنىڭ نىمىنى بىلدۈرىدىغانلىقىنى ئاددىي چۈشەندۈرۈپ ئۆتەي. مىنىڭ كەسپىم كومپيۇتېرغا ئالاقىدار بولغاچقا، تۆۋەندىكىلەرنى ئۆز چۈشەنچەم بىلەن، ناھايىتى ئاددىي تىلدا يېزىشقا تىرىشتىم، ھەم بۇلارنى خەنچە ئۆگەنگەن بولغاچقا ئۇيغۇرچە ئىپادىلىگەندە ئازراق مەسىلە بۆلىشى مۇمكىن، چۈشەنمىگەنلەر ئايرىم ئىنكاس يازارسىلەر.

ئالدى بىلەن گەپنى كومپيۇتېردىن باشلايلى. كومپيۇتېرنىڭ 64 بىتلىق ياكى 32 بىتلىق ئىكەنلىكى مەركىزى بىرتەرەپ قىلغۇچ يەنى CPU ئارقىلىق بەلگىلىنىدۇ. CPU نىڭ سىغدۇرۇشچانلىقى يۇقىرىدىن تۆۋەنگە بولۇپ، 64 بىتلىق CPU ئىشلىتىلگەن كومپيۇتېرلارغا 32 بىتلىق مەشغۇلات سىستېمىسىنىمۇ قاچىلىغىلى بولىدۇ. بىراق 32 بىتلىق CPU ئىشلىتىلگەن كومپيۇتېرلارغا 64 بىتلىق مەشغۇلات سىستېمىسىنىمۇ قاچىلىغىلى بولمايدۇ.

كومپيۇتېر ئىشلىتىۋاتقانلار بىلسە كېرەك، مەركىزى بىرتەرەپ قىلغۇچ (CPU) بىر تەرەپ قىلىدىغان ھەممە ئۇچۇرنى ئىچكى ساقلىغۇچ (内存) دىن ئالىدۇ. ھەمدە بىرتەرەپ قىلىپ بولغان ئۇچۇرنى ئىچكى ساقلىغۇچ (内存) قا ساقلايدۇ، ئاندىن بىز ئىشلىتىمىز. مەركىزى بىرتەرەپ قىلغۇچ (CPU) نىڭ ھەر قېتىم بىر تەرەپ قىلالايدىغان سانلىق قىممەتنىڭ چوڭ كىچىكلىكى سۆز گۇرۇپپىسى (字组/word size) دەپ ئاتىلىدۇ. سۆز گۇرۇپپىسى (字组/word size) نىڭ چوڭ كىچىكلىكى مەركىزى بىرتەرەپ قىلغۇچ (CPU) نىڭ 64 بىتلىق ياكى 32 بىتلىق ئىكەنلىكى تەرىپىدىن بەلگىلىنىدۇ. مەن يۇقىرىدا تىلغا ئالغان كومپيۇتېرنىڭ 64 بىتلىق ياكى 32 بىتلىق ئىكەنلىكى مەركىزى بىرتەرەپ قىلغۇچ (CPU) نىڭ ئانالىز (解析) قىلغان سۆز گۇرۇپپىسى (字组/word size) نىڭ چوڭ كىچىكلىكى تەرىپىدىن بەلگىلىنىدۇ. 32 بىتلىق مەركىزى بىرتەرەپ قىلغۇچ (CPU) ئەڭ چوڭ بولغاندا 4G لىق ئىچكى ساقلىغۇچ (内存) نى قوللايدۇ.

مەركىزى بىرتەرەپ قىلغۇچ (CPU) نىڭ ئىقتىدارى قايسى خىل سىستېمىدا ( 64 بىتلىق ياكى 32 بىتلىق) ئەڭ ياخشى ئىكەنلىكىنى ھېچ قىلدىڭلار ھە قاچان.

2004-يىلىدىن كىيىن چىققان خاتىرە كومپيۇتېر ۋە تەييار ئۈستەل كومپيۇتېرلىرىنىڭ ھەممىسىگە دېگۈدەك 64 بىتلىق مەركىزى بىرتەرەپ قىلغۇچ (CPU) ئىشلىتىلگەچكە، 64 بىتلىق مەشغۇلات سىستېمىسىنى ئەمدى ئىشلىتىپ باقساق بولىدۇ. Windows نىڭ 64 بىتلىقىنى قاچىلىغاندا 32 بىت ئامبىرى سىستېما بىلەن بىرگە سەپلەگلىك بولغاچقا، 32 بىتلىق سىستېما ئۈچۈن ئىشلەنگەن دېتاللارنى 64 بىتلىقى Windows مەشغۇلات سىستېمىسىدا بىمالال ئىشلىتەلەيمىز. ئىككى ئامبار ئايرىم بولغاچقا، 32 بىتلىق 64 بىتلىق سىستېما ئۈچۈن ئىشلەنگەن دېتاللار ئارىلىشىپ كەتمەيدۇ.

ئۇبۇنتۇ ۋە باشقا لىنۇكس مەشغۇلات سىستېمىلىرىنىڭ ISO ھۆججىتىنى ياكى دېتالى ھۆججىتى ئاخىرىغا قوشۇلغان i386، i586، i686 دېگەنلەر بۇ ISO نىڭ 32 بىتلىق كومپيۇتېر ئۈچۈن ئىشلەنگەنلىكىنى ( 64 بىتلىق CPU ئىشلىتىلگەن كومپيۇتېرلارغا 32 بىتلىق مەشغۇلات سىستېمىسىنىمۇ قاچىلىغىلى بولىدۇ ئەلۋەتتە )، ياكى 32 بىتلىق مەشغۇلات سىستېمىسى ئۈچۈن دېتال دېگەننى ئىپادىلەيدۇ.
x86_64، AMD64 بولسا 64 بىتلىق سىستېما ISO ھۆججىتىنى ياكى دېتال دېگەننى ئىپادىلەيدۇ.

ئۇبۇنتۇ 11.04 تىن باشلاپ، 64 بىتلىق ئۇبۇنتۇغا 32 بىت ئامبىرى سەپلەنگەن، ئۇبۇنتۇنىڭ كونا نەشرىنى ئىشلىتىۋاتقانلار، ياكى 32 بىت ئامبىرى بىنورماللار تۆۋەندىكى بۇيرۇق ئارقىلىق قاچىلىساڭلار بولىدۇ.

sudo apt-get install ia32-libs*

32 بىت ئامبىرىنى قاچىلاپ بولغاندىن كىيىن 64 بىتلىق ئۇبۇنتۇدا 32 بىتلىق سىستېما ئۈچۈن ئىشلەنگەن دېتاللارنى قاچىلاپ ئىشلىتەلەيمىز. نىمىلا دېگەن بىلەن 64 بىتلىق سىستېما نۇرغۇن كىشىلەرگە ناتونۇش، دېتاللىرى 32 بىتلىق سىستېمىدىن ئاز.

قېنى ئەمىسە مەن ئىشلىتىلىۋاتقان 64 بىتلىق ئۇبۇنتۇ 11.04 كە، 32 بىتلىق سىستېما ئۈچۈن ئىشلەنگەن google chrome جاھاننامىسىنى قاچىلاپ، بۇ ئامبارنى قانداق ئىشلىتىشنى كۆرسىتىپ ئۆتەي.

مەسىلەن مەن http://www.google.com/chrome بۇ ئۇلىنىشتىن google-chrome-stable_current_i386.deb ھۆججىتىنى چۈشۈردۈم، بۇنى 64 بىتلىق ئۇبۇنتۇغا قاچىلىماقچى، ئۇنداقتا تېرمىنالدا cd بۇيرۇقى ئارقىلىق بايا چۈشۈرگەن ھۆججەت بار مۇندەرىجىگە كىرىپ، تېرمىنالغا تۆۋەندىكى بۇيرۇقنى يېزىپ قاچىلايمەن.
